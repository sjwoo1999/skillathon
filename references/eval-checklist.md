# Evaluation Checklist

Use this checklist before submitting.

## Reproducibility
- [ ] Another user can understand the task from `README.md`.
- [ ] Another user can run the workflow from `SKILL.md`.
- [ ] Mock input data is included.
- [ ] Expected output format is shown.

## Safety
- [ ] No API keys, tokens, webhook URLs, or credentials are present.
- [ ] No private customer data or company internal data is present.
- [ ] Any external service steps require human approval.

## Quality
- [ ] The skill has a narrow trigger condition.
- [ ] Required inputs are listed.
- [ ] Output criteria are clear.
- [ ] Failure or stop conditions are documented.
- [ ] Skipped validations are listed in the final output.
