# 입력 계약

## 필수 입력

- 평가 대상 스킬의 `SKILL.md` 본문 또는 파일 경로
- 사용자가 원하는 평가 범위: 전체 평가, 특정 섹션, 특정 변경분 중 하나

## 선택 입력

- 관련 reference 파일, 예시 프롬프트, 테스트 대화, 사용 로그
- 목표 사용자, 실행 환경, 허용 도구, 금지 행동
- 기존 이슈, 실패 사례, 리뷰 기준의 우선순위
- 사용자가 원하는 엄격도: 기본, 엄격, 릴리스 차단 수준

## 입력이 부족할 때

- 필수 입력이 없으면 평가를 시작하지 말고 짧게 요청한다.
- 선택 입력이 없으면 제공된 자료만으로 제한 평가를 수행한다.
- 범위가 모호하면 가장 좁은 합리적 범위를 가정하고, 가정한 범위를 답변에 명시한다.
- 파일 경로가 주어졌지만 읽을 수 없으면 내용을 붙여 달라고 요청한다.

## 제한 평가 규칙

- 제공된 자료 밖의 동작, 숨은 파일, 실제 도구 실행 결과를 단정하지 않는다.
- 스킬이 요구하는 워크플로, 트리거 조건, 입력 요구사항, 출력 품질, 안전장치만 평가한다.
- 평가 대상이 아닌 코드베이스 품질, 제품 전략, 작성자 의도는 다루지 않는다.
- 사소한 문체 취향보다 실행 실패, 모호한 지시, 과도한 범위, 검증 불가능성을 우선한다.

## 증거와 추론 분리

- `증거`: 문서에 실제로 적힌 문장, 구조, 누락, 충돌을 근거로 든다.
- `추론`: 그 증거가 실행 중 어떤 실패나 혼란으로 이어질 수 있는지 설명한다.
- 추론은 가능성으로 표현하고, 사실처럼 단정하지 않는다.
- 심각도는 증거와 예상 영향이 함께 있을 때만 높인다.
