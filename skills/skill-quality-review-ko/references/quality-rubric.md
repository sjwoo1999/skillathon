# Skill 품질 평가 루브릭

각 항목은 관찰 가능한 증거를 기준으로 `강함`, `보통`, `위험` 중 하나로 평가한다.

## 사용 조건

- 강함: 언제 이 Skill을 써야 하는지와 쓰지 말아야 하는지가 구체적이며, 유사 Skill과의 경계가 분명하다.
- 보통: 주요 사용 상황은 설명하지만 예외, 제외 조건, 우선순위가 일부 모호하다.
- 위험: 적용 범위가 넓거나 추상적이라 잘못 호출될 가능성이 높다.

## 입력 계약

- 강함: 필요한 입력, 선택 입력, 형식, 누락 시 처리 방식이 명시되어 있다.
- 보통: 필요한 정보는 대체로 알 수 있으나 형식이나 누락 대응이 불완전하다.
- 위험: 입력 요구사항이 불명확해 추측에 의존하거나 잘못된 작업을 유발한다.

## 작업 절차

- 강함: 수행 순서가 재현 가능하고, 분기 조건과 도구 사용 기준이 명확하다.
- 보통: 큰 흐름은 있으나 세부 단계, 실패 처리, 반복 조건이 부족하다.
- 위험: 절차가 지시문 수준에 머물러 실행자가 임의로 해석해야 한다.

## 안전성

- 강함: 민감정보, 외부 요청, 파괴적 변경, 승인 필요 작업에 대한 제한이 명시되어 있다.
- 보통: 안전 원칙은 있으나 실제 중단 조건이나 승인 기준이 부족하다.
- 위험: 비밀값 노출, 무단 외부 전송, 되돌릴 수 없는 변경을 막는 지침이 없다.

## 검증 가능성

- 강함: 완료 기준, 확인 방법, 테스트 또는 산출물 검토 방식이 관찰 가능하다.
- 보통: 검증 필요성은 언급하지만 어떤 증거로 판단할지 불명확하다.
- 위험: 결과가 맞는지 확인할 방법이 없거나 주관적 만족에 의존한다.

## 명확성

- 강함: 문장이 짧고 직접적이며, 핵심 용어와 책임 범위가 일관된다.
- 보통: 이해는 가능하지만 중복, 추상 표현, 암묵적 전제가 일부 있다.
- 위험: 모호한 표현이 많아 서로 다른 실행 결과가 쉽게 나온다.

## 확장성/재사용성

- 강함: 다양한 사례에 재사용 가능하며, 예외와 확장 지점이 기존 구조와 잘 맞는다.
- 보통: 현재 사례에는 맞지만 새로운 입력이나 도메인으로 확장할 때 보완이 필요하다.
- 위험: 특정 사례에 과도하게 묶여 있거나 복사 후 수정 없이는 재사용하기 어렵다.
